# Taxi-v3 Q-Learning Project

Учебный проект по Reinforcement Learning: табличный Q-Learning агент обучается
решать среду `Taxi-v3` из Gymnasium без Stable-Baselines3, RLlib и других
готовых RL-библиотек.

Примечание по совместимости: в Gymnasium 1.3+ среда `Taxi-v3` помечена как
устаревшая в пользу `Taxi-v4`. Так как задание требует именно `Taxi-v3`, код
сохраняет `ENV_NAME = "Taxi-v3"` и, если этот ID отсутствует в реестре текущей
Gymnasium, регистрирует его как совместимый alias на встроенную реализацию
`gymnasium.envs.toy_text.taxi:TaxiEnv` с детерминированными настройками по
умолчанию.

```text
taxi_q_learning/
```

```bash
cd taxi_q_learning
python -m venv .venv
.venv\Scripts\activate
python -m pip install --upgrade pip

```

На Linux/macOS активация виртуального окружения будет такой:

```bash

```

## Обучение

```bash
python train.py
```

Скрипт обучает агента, сохраняет `q_table.npy`, строит график
`training_rewards.png` и в конце оценивает greedy-политику на 100 тестовых
эпизодах.

Можно изменить гиперпараметры:

```bash
python train.py --episodes 20000 --alpha 0.1 --gamma 0.95 --epsilon 1.0 --epsilon-decay 0.9995 --min-epsilon 0.05
```

Во время обучения каждые 100 эпизодов выводится средняя награда за последние
100 эпизодов. Это помогает видеть, что агент постепенно переходит от случайного
поведения к более стабильной стратегии.

## Демонстрация обученного агента

## Что такое Reinforcement Learning

Reinforcement Learning, или обучение с подкреплением, - это подход машинного
обучения, в котором агент учится принимать решения через взаимодействие со
средой. Агент наблюдает состояние, выбирает действие, получает награду и
переходит в новое состояние. Цель агента - найти стратегию поведения, которая
максимизирует суммарную награду за эпизод или за долгий горизонт времени.

Классический цикл RL выглядит так:

1. Среда выдает текущее состояние `state`.
2. Агент выбирает действие `action`.
3. Среда возвращает `next_state`, `reward`, `terminated`, `truncated`, `info`.
4. Агент обновляет свои знания.
5. Процесс повторяется до завершения эпизода.

В Gymnasium современный API выглядит так:

```python
state, info = env.reset()
next_state, reward, terminated, truncated, info = env.step(action)
```

`terminated` означает естественное завершение задачи, а `truncated` - досрочное
завершение по внешнему ограничению, например по максимальному числу шагов.

## Что представляет собой Taxi-v3

`Taxi-v3` - это небольшая дискретная среда, где такси движется по сетке 5x5.
Задача агента:

1. Доехать до пассажира.
2. Забрать пассажира действием `pickup`.
3. Доехать до целевой точки.
4. Высадить пассажира действием `dropoff`.

В среде есть 500 дискретных состояний и 6 действий.

Действия:

```text
0 - south, вниз
1 - north, вверх
2 - east, вправо
3 - west, влево
4 - pickup, посадить пассажира
5 - dropoff, высадить пассажира
```

Награды устроены так, чтобы агенту было выгодно быстро доставить пассажира:

```text
+20  за успешную доставку
-10  за неправильный pickup/dropoff
-1   за обычный шаг
```

## State, Action, Reward

`state` - это числовое описание текущей ситуации. В Taxi-v3 состояние кодирует
позицию такси, положение пассажира и пункт назначения.

`action` - это действие, которое агент выбирает в текущем состоянии. Например,
ехать вниз, ехать вправо, посадить или высадить пассажира.

`reward` - это сигнал качества действия. Положительная награда говорит агенту,
что действие было полезным, отрицательная - что действие ухудшило результат или
потратило время.

## Как работает Q-Learning

Q-Learning хранит таблицу `Q[state, action]`. В каждой ячейке находится оценка:
насколько выгодно выполнить конкретное действие в конкретном состоянии.

Сначала Q-таблица заполнена нулями, потому что агент ничего не знает о среде.
Во время обучения агент пробует разные действия, получает награды и обновляет
таблицу. Если действие приводит к хорошему будущему результату, его Q-значение
увеличивается. Если действие приводит к штрафу, его Q-значение уменьшается.

После обучения политика агента становится простой:

```python
best_action = argmax(Q[state])
```

То есть агент выбирает действие с максимальным Q-значением для текущего
состояния.

## Зачем нужен epsilon-greedy

Если агент всегда будет выбирать только лучшее действие по текущей Q-таблице,
он может слишком рано застрять в плохой стратегии. В начале обучения таблица
почти пустая, поэтому агенту нужно исследовать среду.

Epsilon-greedy решает эту проблему:

```text
с вероятностью epsilon     выбираем случайное действие
с вероятностью 1 - epsilon выбираем лучшее действие из Q-таблицы
```

В начале `epsilon` высокий, поэтому агент много исследует. Затем применяется
`epsilon_decay`, и `epsilon` постепенно уменьшается. Агент все чаще использует
накопленный опыт и все реже действует случайно.

## Формула Q-Learning

```text
Q(s_t, a_t) <- Q(s_t, a_t) + alpha * [r_{t+1} + gamma * max_a Q(s_{t+1}, a) - Q(s_t, a_t)]
```

Обозначения:

```text
Q(s_t, a_t)                 текущая оценка качества действия a_t в состоянии s_t
s_t                         текущее состояние
a_t                         выбранное действие
r_{t+1}                     награда после выполнения действия
s_{t+1}                     новое состояние
max_a Q(s_{t+1}, a)         лучшая оценка будущей награды из нового состояния
alpha                       learning rate, скорость обновления таблицы
gamma                       discount factor, важность будущих наград
[ ... ]                     temporal-difference error, ошибка текущей оценки
```

Если новое состояние терминальное, будущая часть `max_a Q(s_{t+1}, a)` считается
равной нулю, потому что после завершения эпизода будущих действий уже нет.

## Как происходит обучение агента

1. Создается среда `Taxi-v3`.
2. Создается Q-таблица размера `500 x 6`.
3. На каждом эпизоде агент начинает из нового случайного состояния.
4. Агент выбирает действие по epsilon-greedy.
5. Среда возвращает новое состояние и награду.
6. Q-таблица обновляется по формуле Q-Learning.
7. После каждого эпизода `epsilon` немного уменьшается.
8. Каждые 100 эпизодов выводится средняя награда.
9. После обучения таблица сохраняется в `q_table.npy`.
10. Строится график `training_rewards.png`.
11. Агент оценивается на 100 тестовых эпизодах без случайных действий.

## Почему агент начинает выбирать лучшие действия

В начале агент часто получает штрафы: ездит не туда, пытается забрать пассажира
не в той клетке или высадить его не в том месте. Эти действия получают низкие
Q-значения. Удачные последовательности действий, которые приводят к доставке,
получают более высокие Q-значения за счет положительной награды `+20` и
дисконтирования будущего результата.

Постепенно Q-таблица становится картой полезности действий. Когда `epsilon`
уменьшается, агент все чаще выбирает действия с максимальным Q-значением и
поэтому действует быстрее и стабильнее.

## Краткий отчет по ожидаемым результатам

После достаточного количества эпизодов средняя награда должна заметно вырасти:
вместо хаотичных отрицательных результатов агент научится доставлять пассажира
за малое число шагов. Успешность на 100 тестовых эпизодах обычно становится
близкой к 100%, а среднее количество шагов снижается, потому что обученная
greedy-политика выбирает маршрут и действия pickup/dropoff более осмысленно.

Фактические числа зависят от версии Gymnasium, seed и числа эпизодов. Для
итогового отчета используйте строки, которые напечатает `python train.py`:

```text
Mean reward
Successful deliveries
Mean steps
```
